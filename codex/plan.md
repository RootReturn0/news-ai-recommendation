开发顺序：
1. 先定义 MVP 产品判断
- 明确“个性化”不是按标签堆新闻，而是根据用户目标、关注点和当前情境，替用户判断哪些新闻值得看
- 选择一个最有辨识度的产品核心，例如“告诉我今天哪些新闻和我的目标最相关，以及为什么”
- 确定 MVP 只服务一个核心场景，避免同时做太多能力

2. 搭建 FastAPI 服务骨架
- 建立基础项目结构，例如 `app/main.py`、`app/routes/`、`app/services/`、`app/models/`
- 提供健康检查接口，确认服务可启动
- 预留 Telegram webhook 或消息入口接口

3. 接入 Telegram Bot 交互层
- 创建 Bot 命令和基础消息处理流程
- 设计首轮对话，快速收集用户偏好、目标或当前关注主题
- 定义最小交互闭环：用户发起请求 -> Bot 收集上下文 -> 返回个性化新闻结果
- 如果时间紧，优先采用最简单稳定的接入方式

4. 封装 OpenAI 调用层
- 使用 OpenAI 的框架或 SDK 建立统一 LLM 调用入口
- 设计 prompt 结构，区分用户画像提取、新闻筛选、结果总结三个任务
- 控制输出格式，尽量返回结构化结果，方便后续拼装 Telegram 消息
- 优先保证调用稳定和结果可解释，不追求复杂 Agent 编排

5. 实现新闻输入层
- 第一优先级是用 `data/rss` 目录下的 RSS 文件作为 mock data 跑通完整链路
- 先从现有样例文件中解析新闻，模拟真实新闻源输入
- 若时间允许，再接入一个简单真实新闻源，例如 RSS 或单一 API
- 统一新闻数据结构，至少包含标题、摘要、来源、链接、时间

6. 实现个性化核心逻辑
- 根据用户输入提取偏好、目标和排除项
- 对新闻做相关性判断、排序和筛选
- 为每条推荐生成“为什么推荐给你”的解释
- 输出结果时避免只给摘要，要体现 Agent 的判断

7. 优化 Telegram 结果呈现
- 设计简洁稳定的消息格式，控制长度，保证手机端易读
- 每次返回有限条新闻，例如 3 到 5 条
- 增加可选反馈信号，例如“想看更多”“少看这类”“只看某个方向”
- 优先保证第一次返回结果足够惊艳、足够清楚

8. 准备 Demo 路径
